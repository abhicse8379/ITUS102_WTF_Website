Lab Tools & Software Guide

What to install, and when

Everything under Core Development Environment should be installed before your first lab session. Everything else can wait until you reach the corresponding unit — installing it all on day one just gives you more things to troubleshoot before you need any of them.

1 · Core Development Environment

INSTALL FIRST
VS Code
All sessionscode.visualstudio.com

Primary code editor for HTML, CSS, JavaScript, and backend code.

  • Install the "Live Server" extension to auto-refresh pages as you edit.
  • Install "Prettier" for automatic code formatting.
  • Use the integrated terminal (Ctrl+`) to run npm/node/git without leaving the editor.
Chrome (or any modern browser)
All sessionsgoogle.com/chrome

Runs and displays your pages; DevTools are used constantly for inspection and debugging.

  • Open DevTools with F12 or Ctrl+Shift+I.
  • Elements tab — inspect and live-edit HTML/CSS.
  • Console tab — view JS errors and test snippets.
  • Network tab — inspect HTTP requests/responses (Units 1, 3, 4).
  • Lighthouse tab — audit performance, accessibility, best practices (Unit 6, Tutorial CO3).
Node.js & npm
Sessions 3–11 (Units 2–4), Tutorial T10–T15nodejs.org

JavaScript runtime and package manager; runs server-side JS and installs libraries/frameworks.

  • Verify with node -v and npm -v in a terminal.
  • npm init -y to start a new project.
  • npm install <package> to add a library (e.g. express).
Git & GitHub
Session 2 (Unit 1) onward, all lab submissionsgit-scm.com

Version control for tracking changes and submitting/sharing lab work.

  • git init to start tracking a project folder.
  • git add . then git commit -m "message" to save a checkpoint.
  • git push to upload work to a GitHub repository — create a free account at github.com.

2 · Frontend Styling & UI Tools

CO2 · CO3 · CO4
Bootstrap
Tutorial T10–T12, capstone projectgetbootstrap.com

CSS framework with ready-made responsive components — navbars, cards, grids, forms.

  • Add via CDN <link>/<script> tags — no install needed for basic use.
  • Use the 12-column grid (row/col-*) to build responsive layouts fast.
  • Browse the official docs for ready-to-copy component snippets.
Tailwind CSS
Tutorial T10–T12, capstone projecttailwindcss.com

Utility-first CSS framework for custom responsive designs without separate CSS files.

  • Quickest start: the Tailwind CDN script tag for practice/lab use.
  • Style directly with utility classes, e.g. class="flex p-4 md:p-8".
  • Use responsive prefixes (sm:, md:, lg:) to adapt per screen size.
WAVE Accessibility Checker
Tutorial T7–T9 (CO3)wave.webaim.org

Browser-based tool that flags accessibility issues — missing alt text, contrast, labels.

  • Install the WAVE browser extension, click it while viewing your page.
  • Fix flagged errors first (red icons), then review alerts (yellow icons).
  • Re-run after each fix to confirm resolution.

3 · Backend & API Tools

Unit 3
Express.js
Sessions 7–9 (Unit 3)expressjs.com

Minimal Node.js framework for building server-side routes and APIs.

  • Install with npm install express.
  • Start a server with app.listen(PORT); define routes with app.get()/app.post().
  • Use nodemon (npm install -g nodemon) so the server auto-restarts on file changes.
Postman (or Thunder Client)
Session 1 (Unit 1), Sessions 7–9 (Unit 3)postman.com

Sends HTTP requests to test your APIs without needing a front-end.

  • Create a request, set the method (GET/POST/PUT/DELETE) and URL.
  • Check the response body, headers, and status code returned.
  • Save related requests in a "collection" for reuse. Thunder Client is a lightweight VS Code-extension alternative.

4 · Database Tools

Unit 4

Your instructor will confirm whether the lab uses the MySQL or MongoDB stack — you likely won't need both.

MySQL Workbench
Sessions 10–11 (Unit 4)dev.mysql.com

Visual tool to design, query, and manage a MySQL relational database.

  • Connect to your local MySQL server using the connection wizard.
  • Use the SQL editor to run CRUD queries (SELECT/INSERT/UPDATE/DELETE).
  • Use the schema browser to inspect tables without writing SQL.
MongoDB Compass
Sessions 10–11, if using a NoSQL stackmongodb.com/products/compass

GUI for viewing and querying data in a MongoDB (NoSQL) database.

  • Connect using a local connection string, e.g. mongodb://localhost:27017.
  • Browse collections and documents visually, or use the query bar.
  • Use the "Explain Plan" tab to see query performance — ties into query-optimization awareness.

5 · Security & Testing Awareness Tools

Unit 5

For inspecting and understanding your own lab projects only.

Chrome DevTools — Security & Application
Sessions 12–13 (Unit 5)Built into Chrome

Inspect HTTPS/certificate status, cookies, local/session storage, and stored credentials.

  • Security tab — check whether a page is served over a valid HTTPS connection.
  • Application tab — inspect cookies and storage to see how session data is kept client-side.
  • Never enter real passwords into test pages while exploring these tabs.
OWASP ZAP
Sessions 12–13, instructor-guided demozaproxy.org

Beginner-friendly scanner for common web-application misconfigurations — awareness-level use only.

  • Only ever scan applications you own or have explicit permission to test.
  • Run "Automated Scan" against your own local lab project to see typical findings.
  • Review the generated report to understand vulnerability categories, not to exploit them.

Getting help

If a tool fails to install or behaves unexpectedly: note the exact error message and bring it to your next lab or tutorial session. Most setup issues are quick to resolve with the instructor or TA present.
